# **Ansible-playbook для установки и настройки Wireguard и модуля xt_wgobfs**
Плейбук устанавливает и настраивает Wireguard и модуль ядра Linux xt_wgobfs, который служит для обфускации Wireguard-трафика, на сервере и клиенте на ОС Debian. 

**Плейбук автоматически выполняет:** 

1. Генерирование параметров: пары ключей для сервера и клиента, порта сервера и ключа обфускации. 
2. Установку и настройку Wireguard, применение сгенерированных параметров на WG-сервере и WG-клиенте.
3. Установку и настройку модуля ядра xt_wgobfs на WG-сервере и WG-клиенте.
4. Установку iptables, добавление правил для обвускации трафика. 
5. Ping WG-интерфейса клиента с сервера, что бы убедиться, что туннель работает. 

 

# **Состав проекта**

1. **main.yml** - главный плейбук, который автоматически создаёт пару сервер-клиент. 
2. **wgServerInstall.yml** - генерирует параметры, устанавливает и настраивает WG-сервер и модуль xt_wgobfs.
3.  **wgClientInstall.yml** - устанавливает и настраивает WG-клиент и модуль xt_wgobfs. Параметры должны быть либо указаны в файле group_vars/serverParameters.yml либо переданы через аргументы командной строки (ansible-playbook --extra-vars).
4. **playbooks/:**
    1. **generateVariables.yml** - генерирует параметры для конфигурации WG и xt_wgobfs и записывает их в файл group_vars/serverParameters.yml в виде переменных. Запускается на localhost, устанавливает пакет wireguard-tools для генерации пар ключей.
    2. **ping.yml** - пингует WG-интерфейс клиента.
5. **group_vars/:**
    1. **serverParameters.yml** - параметры для конфигурации WG и xt_wgobfs: порт WG-сервера, пары ключей, ключ для обфускации.
    2. **wgIntfName_IP.yml** - имена и IP wg-интерфейсов сервера и клиента.
6. **roles/**
    1. **wireguardServer** - установка и настройка WG-сервера и xt_wgobfs.
    2. **wireguardClient** - установка и настройка WG-клиента и xt_wgobfs.

# **Использование**

В файл inventory/inventory.ini внести данные для подключения к хостам.

Имена и IP WG-интерфейсов можно установить в файле group_vars/wgIntfName_IP.yml или передать через аргументы командной строки. По умолчанию имя WG-интерфейса - wg0, IP-адрес - 10.0.0.1/24 на сервере; 10.0.0.2/24 на клиенте.  

**Автоматическое создание пары сервер-клиент**

main.yml — автоматически сгенерирует все необходимые параметры для конфигурации, запустит установку и настройку WG-сервера и WG-клиента с модулем для обвускации трафика xt_wgobfs. В конце будет отправлено 10 ping-запросов через туннель для проверки работоспособности. 

**`ansible-playbook main.yml`** 

**Установка и настройка сервера и клиента отдельно**

wgServerInstall.yml — атомический сгенерирует все необходимые параметры для конфигурации, запустит установку и настройку WG-сервера с модулем для обвускации трафика xt_wgobfs. В конце будет выведена информация: порт сервера, публичный ключ сервера и приватный ключ клиента, ключ обфускации для дальнейшей настройки WG-клиента.

**`ansible-playbook wgServerInstall.yml`**

wgClientInstall.yml — запустит установку и настройку WG-клиента с модулем для обвускации трафика xt_wgobfs. Параметры (порт WG-сервера, пары ключей, ключ для обвускации) должны быть указаны в файле group_vars/serverParameters.yml или переданы через аргументы командной строки.

**`ansible-playbook wgClientInstall.yml`**

Для создания нескольких WG-интерфейсов на сервере или клиенте, перед каждым последующим запуском сценария нужно изменять имена и IP WG-интерфейсов в файле group_vars/wgIntfName_IP.yml или передавать значения через аргументы командной строки.

**`ansible-playbook wgServerInstall.yml -e "serverWgIntfName=wg1" -e "serverWgIntfIP=10.0.1.1”`**

**`ansible-playbook wgClientInstall.yml -e "clientWgIntfName=wg1" -e "clientWgIntfIP=10.0.1.2”`**
